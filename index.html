Palette Settings

What is Chroma factor?

The Chroma factor decreases or increases the overall saturation of colors. The base chroma value is "?".

"1" is the calculated maximum safe chroma value.
"0" is grayscale.
The value can be increase to up to "2". But for most hues this looks way oversaturated.

Dynamic Hue Shift

What is bright colors mix hue?

Brighter colors get shifted towards this hue. The brighter the color the stronger the shift towards this hue. For warmer shifts this value should be around "80".

What is dark colors mix hue?

Darker colors get shifted towards this hue. The darker the color the stronger the shift towards this hue. For colder shifts this value should be around "260".

Show Base Color Values

What are base color values?

These values represent the lightness, chroma, and hue values of the base color. That's the color before Dynamic Hue Shift gets applied.

Currently, final colors get generated using the CSS function color-mix(). AFAIK I cannot extract those values from colors generated using that function.

About

This is a playground where I test if it's possible to generate automatic color palettes using the OKLCH color space and some calculations on top of it.

At the moment, palettes cannot be created in a form that's usable in CSS or other software. That's because the CSS function color-mix() is used to generate the final individual colors.

The goal is to wait if and when the CSS proposals @function and @mixin get finalized and implemented by browsers. If that's the case entire palettes can be configured and generated inside of CSS.

How do colors for a hue get generated?

I start with a hue value. I then have a function that turns a base chroma value (which is "?") into one that depends on an individual lightness value. That means for lightnesses "0" and "1" it gives me a chroma value of "0". For the lightness value "?" it gives me the chroma value "1". Everything in between gets interpolated linearly.

I've also experimented with using this value ("?") to generate the max chroma for each individual hue at this lightness value. That would result in more saturated colors for hues that support it. But I wasn't satisfied with how it turned out. It results in mostly very saturated pinks, but not yellows, because yellow is most saturated at a very high lightness value. That would require much more complex formulas. In addition to that I need to apply those chroma variations in moderation. Otherwise I guess the palettes look more like they are based on the HSL color space, which is not my goal.
